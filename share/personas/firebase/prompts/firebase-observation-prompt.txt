Here's what the documentation should look like:

---

# Fun Card Reload API

## What It Does
Adds funds to a guest's Fun Card. That's it. Nothing else.

## Endpoint
```
POST /api/v1/funcard/reload
```

## Authentication
Required. Firebase ID token in Authorization header.

No token? You get 401. Wrong user's card? You get 403.

## Request Body
- cardId: string (required) - The Fun Card ID
- amount: number (required) - Dollars to add, min 5, max 500
- paymentMethodId: string (required) - Stripe payment method

## Important Notes
- Minimum reload: $5
- Maximum reload: $500
- Processing time: Usually <3 seconds
- If timeout, CHECK BALANCE FIRST before retrying (avoid double-charge)

---

That's it. Clear, direct, actionable. No more confusion about how this endpoint works."
