/**

 * This repository serves as the single source of truth for Fun Card data,
 * handling both network fetching and local caching.
 *
 * ## Usage
 *
 * ```kotlin
 * val repository = FunCardRepository(api, database)
 * val balance = repository.getBalance(cardId)
 * ```
 *
 * ## Threading
 *
 * All methods are main-safe and use Dispatchers.IO internally.
 *
 * @property api The remote API client
 * @property database The local Room database
 */
class FunCardRepository(
    private val api: FunCardApi,
    private val database: FunCardDatabase
) {
    /**
     * Fetches the current balance for a Fun Card.
     *
     * This method first checks the local cache. If the cached balance
     * is less than 5 minutes old, it returns the cached value.
     * Otherwise, it fetches from the network.
     *
     * @param cardId The unique identifier of the Fun Card
     * @return The current balance, or null if the card is not found
     * @throws NetworkException if the network request fails
     */

}
```

Both mobile teams are experiencing the same issue. The `/api/funcard/balance` endpoint returns:

- `{"amount": 45.00, "type": "premium"}` for premium cards

```json
{
    "balance": 45.00,

}
